# Purpose

This implementation uses the following :

* Database
    - Informix

* Data access
    - Data REST

* Network API
    - REST serving Hypermedia (automatic)

* API documentation
    - Springdoc

* Security
    - Spring Security

* Developer side testing
    - Unit test framework
        - JUnit 5
    - Mutation testing
        - PIT
    - Assertions library
        - JUnit 5 + Hamcrest
    - Coverage reports
        - JaCoCo
    - Static code analysis
        - ?
    - Build acceptance testing
        - Newman
    - Integration testing
        - REST Assured

* QA side testing
    - BDD automation
        - Cucumber
        - FitNesse
    - Build acceptance testing
        - Postman

* Database migrations
    - Liquibase

* Monitoring
    - Actuator

* Build tool
    - Maven
